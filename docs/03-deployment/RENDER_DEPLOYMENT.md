## Step 1: Database Infrastructure Setup

### 1.1 Create Render PostgreSQL Database

1. **Navigate to Render Dashboard**
   - Go to [render.com](https://render.com) and sign in
   - Click "New +" → "PostgreSQL"

2. **Database Configuration**
   ```
   Name: runhouston-db
   Database: runhouston
   User: rh_user
   Region: US East (or closest to your users)
   Plan: Starter ($7/month) → Upgrade to Production ($25/month) for production
   ```

3. **PostGIS Extensions Setup**
   - After database creation, connect to the database
   - Run the following SQL to enable PostGIS extensions:
   ```sql
   CREATE EXTENSION IF NOT EXISTS postgis;
   CREATE EXTENSION IF NOT EXISTS postgis_topology;
   CREATE EXTENSION IF NOT EXISTS postgis_sfcgal;
   ```

4. **Environment Variables** (Auto-generated by Render)
   - `DATABASE_URL`: Auto-generated connection string
   - `POSTGRES_HOST`: Database hostname
   - `POSTGRES_PORT`: 5432
   - `POSTGRES_DB`: runhouston
   - `POSTGRES_USER`: rh_user
   - `POSTGRES_PASSWORD`: Auto-generated password

### 1.2 Database Migration

Once the database is created, run the migration script to set up the schema:

```bash
# Set environment variables for production
export DATABASE_URL="your_render_database_url_here"

# Run migrations
python scripts/migrate.py --env prod --verbose
```

**Migration file that will be applied:**
- `20250909_2026_complete_database_schema.sql` - Complete database schema setup

This single migration file includes:
- PostGIS extensions setup
- All database tables (races, race_reports, clubs, schema_migrations)
- Spatial indexes and triggers
- Admin user creation
- All necessary functions and constraints

## Step 5: Domain and SSL Configuration

### 5.1 DNS Configuration

Configure your DNS records to point to Render:

```
# Main app domain
runhouston.app        → CNAME runhouston-main.onrender.com
www.runhouston.app    → CNAME runhouston-main.onrender.com

# Admin app domain  
admin.runhouston.app  → CNAME runhouston-admin.onrender.com

# API domain
api.runhouston.app    → CNAME runhouston-api.onrender.com
```

### 5.2 SSL Certificate Setup

1. **Automatic SSL** (Recommended)
   - Render automatically provisions Let's Encrypt certificates
   - SSL certificates are renewed automatically
   - HTTP to HTTPS redirect is enabled by default

2. **Custom SSL** (Optional)
   - Upload your own SSL certificate if needed
   - Configure custom certificate in Render dashboard

### 5.3 Domain Verification

1. **Add Custom Domains** in Render Dashboard:
   - Go to each service settings
   - Add custom domain
   - Verify DNS configuration

2. **SSL Certificate Status**:
   - Wait for SSL certificate provisioning (usually 5-10 minutes)
   - Verify HTTPS is working for all domains
